This guide walks you through installing Antfly, loading data, and running your first queries — from full-text search to semantic search to AI-powered RAG.

## Install Antfly

Visit the [Downloads page](/docs/downloads) and follow the instructions for your platform.

## Start Antfly

Pull a text embedding model for semantic search. [Termite](/docs/guides/termite) is Antfly's built-in ML inference engine — it runs ONNX-optimized models locally for fast CPU inference, no external APIs needed.

```bash
antfly termite pull --variants i8 BAAI/bge-small-en-v1.5
```

Start Antfly in swarm mode:

```bash
antfly swarm 2>&1 | tee "antfly.log"
```

<Callout type="info">
Swarm mode starts metadata, data nodes, and Termite in a single process — ideal for local development. You can also access the [web dashboard](/docs/guides/dashboard) at `http://localhost:8080` to manage tables, run queries, and monitor your cluster.
</Callout>

## Download Sample Data

In a new terminal, download 10,000 Wikipedia articles enriched with thumbnail images:

```bash
curl -L -o wiki-articles.json https://cdn.antfly.io/datasets/wiki-articles-10k-v001.json
head -n 1 wiki-articles.json | jq '.'
```

Each article has `title`, `body`, `url`, and `thumbnail_url` fields.

## Create a Table

Create a table with a text embedding index for semantic search:

```bash
antfly table create --table wikipedia \
  --index '{
    "name": "title_body",
    "type": "embeddings",
    "template": "{{title}} {{body}}",
    "embedder": {
      "provider": "termite",
      "model": "BAAI/bge-small-en-v1.5"
    },
    "chunker": {
      "provider": "antfly",
      "text": {
        "target_tokens": 200,
        "overlap_tokens": 25
      }
    }
  }'
```

Verify the table and index were created:

```bash
antfly table list
antfly index list --table wikipedia
```

## Load Data

```bash
# Load all 10,000 articles using the title as the document ID
antfly load --table wikipedia \
  --file wiki-articles.json \
  --id-field title
```

You can monitor embedding progress with:

```bash
antfly index list --table wikipedia
```

## Query the Data

<Callout type="info">
You can run these queries using the CLI (shown below) or the web dashboard at `http://localhost:8080`.
</Callout>

### Full-Text Search

Search for articles containing specific terms (a full specification of the query syntax is available in the [Bleve Query String Query documentation](https://blevesearch.com/docs/Query-String-Query/)):

```bash
# Search for articles about Korea
antfly query --table wikipedia \
  --full-text-search 'body:"Korea"' \
  --fields "title,url" \
  --limit 5
```

### Semantic Similarity Search

Find articles semantically similar to a query:

```bash
# Find articles about anatomy and physiology
antfly query --table wikipedia \
  --semantic-search "anatomy and physiology" \
  --indexes "title_body" \
  --fields "title,url" \
  --limit 5
```

### Hybrid Search

Combine full-text and vector search with neural reranking for better relevance.

Pull the reranker model:

```bash
antfly termite pull --variants i8 mixedbread-ai/mxbai-rerank-base-v1
```

<Callout type="info">
If Antfly doesn't detect the new model, restart `antfly swarm`.
</Callout>

```bash
# Find articles about Einstein with semantic understanding
antfly query --table wikipedia \
  --full-text-search 'body:Einstein' \
  --semantic-search "theory of relativity and physics" \
  --indexes "title_body" \
  --fields "title,url" \
  --limit 10 \
  --reranker '{
      "provider": "termite",
      "model": "mixedbread-ai/mxbai-rerank-base-v1",
      "field": "body"
    }' \
  --pruner '{"min_score_ratio": 0.01}'
```

<Callout type="info">
**Reranking vs Pruning:**
- **Reranking** uses a cross-encoder model (like `mixedbread-ai/mxbai-rerank-base-v1`) to re-score results based on query-document relevance. It improves ordering but keeps the same number of results.
- **Pruning** filters out low-relevance results based on score quality. Use `min_score_ratio` to keep only results scoring at least N% of the top result, or `max_score_gap_percent` to detect "elbows" in score distribution.
</Callout>

## Image Search

Search by visual content using CLIP, a multimodal embedding model that understands both text descriptions and images.

<Steps>
### Pull the CLIP model

```bash
antfly termite pull --variants i8 openai/clip-vit-base-patch32
```

### Add a CLIP index to the table

```bash
antfly index create --table wikipedia \
  --index thumbnail \
  --type embeddings \
  --template "{{media url=thumbnail_url}}" \
  --embedder '{"provider": "termite", "model": "openai/clip-vit-base-patch32"}' \
  --dimension 512
```

### Search by image content

CLIP understands visual concepts, so you can describe what you're looking for:

```bash
# Find articles with images of maps or geography
antfly query --table wikipedia \
  --semantic-search "map of a country" \
  --indexes "thumbnail" \
  --fields "title,url,thumbnail_url" \
  --limit 5
```
</Steps>

<Callout type="info">
CLIP responds best to concrete visual descriptions ("red sports car", "mountain landscape") rather than abstract concepts. Articles without a `thumbnail_url` are skipped by the CLIP index. For a deeper dive, see the [Image Search with CLIP](/docs/examples/image-search) example.
</Callout>

### Multi-Index Hybrid Search

Combine text, image, and full-text search across multiple indexes:

```bash
# Search with both text and image indexes
antfly query --table wikipedia \
  --full-text-search 'title:History' \
  --semantic-search "ancient civilizations and archaeology" \
  --indexes "title_body,thumbnail" \
  --fields "title,url,thumbnail_url" \
  --limit 10
```

## RAG (Retrieval-Augmented Generation)

Combine search with LLM-powered classification and generation using the retrieval agent.

<Steps>
### Classify a query

Pull a small generator model and use it to classify whether a query needs retrieval:

```bash
antfly termite pull hf:onnx-community/gemma-3-270m-it-ONNX
```

```bash
antfly agents retrieval --table wikipedia \
  --semantic-search "What are the major events in Korean history?" \
  --indexes "title_body" \
  --fields "title,body" \
  --limit 5 \
  --reranker '{
      "provider": "termite",
      "model": "mixedbread-ai/mxbai-rerank-base-v1",
      "field": "body"
    }' \
  --pruner '{"min_score_ratio": 0.6, "max_score_gap_percent": 40}' \
  --generator '{
    "provider": "termite",
    "model": "onnx-community/gemma-3-270m-it-ONNX"
  }' \
  --max-context-tokens 512 \
  --classify
```

This command:
1. Searches for semantically similar articles using the generated embeddings
2. Reranks results using the built-in cross-encoder reranker for better relevance
3. Prunes low-quality results (keeps only those scoring ≥60% of top result, stops at large score gaps)
4. Fits the remaining documents into a 512-token context budget, dropping lowest-ranked documents that don't fit
5. Classifies the query intent using the small Gemma 3 model

<Callout type="tip">
**Pruning for RAG:** Pruning is especially valuable for RAG pipelines. Score-based pruning (`--pruner`) filters out marginally relevant documents, while token-based pruning (`--max-context-tokens`) ensures the context fits within the model's limits.
</Callout>

For full generation features (`--generate`, `--reasoning`, `--followup`), you need a larger model. Choose a provider below:

<Callout type="info">
**Choosing a generator model:**
- **`onnx-community/gemma-3-270m-it-ONNX`** (~270M params, 1.1 GB) — fast, runs anywhere, good for `--classify` only
- **`onnxruntime/Gemma-3-ONNX`** (~4B params, 5.7 GB) — local ONNX inference via Termite, supports all RAG features
- **Ollama `gemma3:4b-it-qat`** — local inference via Ollama, quantized for fast CPU/GPU performance
- **Gemini `gemini-2.5-flash`** — cloud API, fast and high quality, requires API key
- **OpenAI `gpt-4.1-mini`** — cloud API, fast and high quality, requires API key
</Callout>

### RAG with structured output

Add `--streaming=false` to get structured JSON output with source references instead of streaming text. This works with any of the generator providers above:

```bash
antfly agents retrieval --table wikipedia \
  --semantic-search "Explain the theory of relativity" \
  --indexes "title_body" \
  --fields "title,body,url" \
  --limit 5 \
  --generator '{"provider": "gemini", "model": "gemini-2.5-flash", "api_key": "${GEMINI_API_KEY}"}' \
  --streaming=false --generate
```

### RAG with evaluation

Add `--eval` to run evaluators on the query results and include scores in the response. Evaluation requires a model capable of generation — use a larger model as the judge:

```bash
antfly agents retrieval --table wikipedia \
  --semantic-search "What are the major events in Korean history?" \
  --indexes "title_body" \
  --fields "title,body" \
  --limit 5 \
  --generator '{"provider": "gemini", "model": "gemini-2.5-flash", "api_key": "${GEMINI_API_KEY}"}' \
  --generate \
  --eval '{
    "evaluators": ["faithfulness", "relevance"],
    "judge": {"provider": "gemini", "model": "gemini-3.0-flash", "api_key": "${GEMINI_API_KEY}"}
  }'
```

Available evaluators:
- **Retrieval metrics** (require `ground_truth.relevant_ids`): `recall`, `precision`, `ndcg`, `mrr`, `map`
- **LLM-as-judge metrics** (require `judge` config): `relevance`, `faithfulness`, `completeness`, `coherence`, `safety`, `helpfulness`, `correctness`, `citation_quality`
</Steps>

## Next Steps

1. **Image Search Example**: Build a full image search app with CLIP — see the [Image Search with CLIP](/docs/examples/image-search) example
2. **Explore Different Models**: Browse available Termite models with `antfly termite list --remote` — try `mixedbread-ai/mxbai-embed-large-v1` for higher quality text embeddings or `mixedbread-ai/mxbai-rerank-base-v1` for neural reranking
3. **Backup and Restore**: Try backing up your data:
```bash
antfly backup --table wikipedia \
  --backup-id wiki-backup-$(date +%Y%m%d) \
  --location "file:///tmp/antfly_backups"
```
4. **Production Deployment**: For production, run metadata and storage nodes separately for better scalability
